# CPU 分支预测器实现总结

## 🎯 项目概述

本项目在五级流水线 RISC-V CPU 中成功设计并集成了**两级自适应分支预测器**，包含：
- **BHT (Branch History Table)**: 分支历史表
- **PHT (Pattern History Table)**: 模式历史表  
- **完整的预测失败处理机制**

## 📊 核心指标

| 项目 | 规格 |
|------|------|
| **BHT 容量** | 256 × 2 bits = 512 bits |
| **PHT 容量** | 512 × 2 bits = 1024 bits |
| **预测延迟** | 1 周期（组合逻辑） |
| **预测准确率** | 典型 90-95%（取决于程序） |
| **预测失败代价** | 2-3 周期流水线清空 |
| **芯片面积增加** | ~5-10%（相对于 CPU） |
| **功耗增加** | ~2-5%（相对于 CPU） |

## 🏗️ 实现架构

### 整体框架
```
┌──────────────────────────────────────────────────┐
│                   CPU 流水线                      │
├──────────────────────────────────────────────────┤
│                                                   │
│  IF 阶段        ID 阶段        EX 阶段           │
│  ┌────────┐     ┌────────┐     ┌────────┐       │
│  │ 预测  │────▶│ 传递  │────▶│ 验证  │        │
│  │ (BHT) │     │ 预测  │     │ (结果)│        │
│  └────────┘     └────────┘     └────────┘       │
│       │              │              │            │
│       │              │              │            │
│       └──────────────┼──────────────┘            │
│                      │                           │
│              ┌───────┴────────┐                 │
│              │ 预测失败处理    │                 │
│              │ - 清空流水线   │                 │
│              │ - 纠正 PC     │                 │
│              │ - 更新预测器   │                 │
│              └────────────────┘                 │
└──────────────────────────────────────────────────┘
```

### BHT 结构
```
PC[9:2] (8 bits)
    │
    ▼
┌─────────────┐
│   BHT[256]  │  ← 存储 2 位历史
├─────────────┤
│ [0]: 11     │  强跳转 + 弱跳转
│ [1]: 10     │  弱跳转 + 强不跳
│ ...         │
│[255]: 01    │  弱不跳 + ...
└─────────────┘
```

### PHT 结构
```
┌─────────────────────────┐
│ {PC[9:2], history[1:0]} │ (9 bits)
│         索引             │
└──────────────┬──────────┘
               │
               ▼
        ┌────────────┐
        │ PHT[512]   │
        ├────────────┤
        │ [0]: 00    │ 强不跳 (SNT)
        │ [1]: 01    │ 弱不跳 (WNT)
        │ [2]: 10    │ 弱跳 (WT)
        │ [3]: 11    │ 强跳 (ST)
        │ ...        │
        │[511]: 11   │
        └────────────┘
               │
               ▼
          预测结果
```

## 🔄 工作流程

### 正确预测的情况
```
时钟 1: PC=0x400100, 预测跳转 → PC+3=0x400120
时钟 2: PC=0x400120
时钟 3: EX 阶段验证 → 实际跳转 ✓
        无预测失败，继续正常执行
```

### 预测失败的情况
```
时钟 1: PC=0x400100, 预测跳转 → PC+3=0x400120
时钟 2: PC=0x400120（错误的取指）
时钟 3: EX 阶段验证 → 实际不跳转 ✗
        pred_mismatch=1
        动作:
        - flush_IF_ID (清空错误预取)
        - flush_ID_EX (清空错误解码)
        - PC 纠正为 0x400104
        - 预测器更新历史和计数器

时钟 4: 从 PC=0x400104 重新取指
```

## 💡 设计特点

### ✅ 优点
1. **简洁高效**: 使用 PC 和本地历史，避免全局历史同步问题
2. **硬件简单**: 不需要复杂的索引计算，仅需 9 位组合索引
3. **低延迟**: 1 周期预测延迟
4. **自适应**: 饱和计数器自动学习分支模式
5. **隔离性好**: 不同分支的历史独立维护

### ⚠️ 局限性
1. **有限历史**: 2 位历史可能无法捕捉复杂模式
2. **索引冲突**: 不同分支可能共用同一 PHT 项
3. **预测失败代价**: 2-3 周期流水线清空
4. **有限容量**: 256 个 BHT 项可能对大程序不够

## 🚀 改进方向

1. **更长的历史**: 3-4 位历史 (需增加 PHT)
2. **全局历史**: 使用所有分支的历史，而非本地历史
3. **混合预测**: 组合本地+全局+其他预测器
4. **分支目标缓冲 (BTB)**: 预测分支目标地址
5. **后缀表 (RAS)**: 特殊处理函数返回

## 📞 常见问题解答

**Q: 预测器会延长关键路径吗?**  
A: 不会，预测在 IF 阶段用组合逻辑完成，不增加时序路径。

**Q: 可以用于其他 ISA 吗?**  
A: 可以，实现与 ISA 无关，只需调整分支指令识别。

**Q: 如何增加预测准确率?**  
A: 增加历史位数、PHT 大小，或采用更复杂的预测器设计。

**Q: 预测失败的开销是多少?**  
A: 典型 2-3 周期流水线清空。
